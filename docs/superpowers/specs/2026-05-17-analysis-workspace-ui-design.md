# 分析工作台 UI 改版 — 设计规格

## 目标

重新设计分析工作台页面，使其视觉更精致，带有标签页报告查看器、流畅的动画和更好的加载状态。当前 UI 只是把 markdown 原文直接塞进卡片 — 新设计应该像一个专业的内容分析仪表盘。

## 架构

单页 vanilla JS SPA（不引入框架）。在现有 `analysis.js` 和 `style.css` 基础上增强。通过 CDN 引入 Open Props 作为设计 token。所有现有 API 接口不变。

## 组件

### 1. 视频列表卡片

**当前：** 简单卡片，显示文件名 + 状态标签 + 文件大小。

**新设计：**

- 左侧彩色边框表示状态：
  - 绿色（`--accent`）= 已分析
  - 蓝色（`--accent-secondary`）= 分析中
  - 黄色（`--warning`）= 等待中
  - 灰色（`--text-secondary`）= 未分析
- 保留状态标签（文字 + 颜色）以确保可访问性（颜色不是唯一指示）
- 悬停：轻微上浮（`transform: translateY(-2px)`）+ 边框发光
- 选中/展开状态：绿色高亮边框，背景稍亮
- 桌面 3 列网格（≥1024px），平板 2 列（768-1023px），手机 1 列（<768px）
- 交错入场动画：淡入 + 上滑，每张卡片延迟 50ms

### 2. 可展开的报告卡片

**触发：** 点击视频卡片。

**行为：**
- 被点击的卡片向下展开，平滑高度动画（300ms ease-out）
- 展开区域使用 `--bg-deep` 背景，与网格视觉分离
- 展开时其他卡片变暗至 opacity 0.6
- 右上角关闭按钮（X）可收起
- 同时只能展开一张卡片（点击另一张会收起当前的）

**展开区域内部布局：**
- 头部：视频文件名（Fira Code）+ 状态标签 + 关闭按钮
- 头部下方是标签栏
- 标签内容区域有内边距

### 3. 标签页报告分段

**"Full 分析"模式（4 个标签）：**

| 标签 | 内容 | Markdown 段落 |
|-----|---------|-------------------|
| 摘要 | 内容摘要 + 核心观点 | 第 1-2 节 |
| 评分 | Rubric 打分 + 综合得分 | 第 3-4 节 |
| 预测 | 爆款预测 | 第 5 节 |
| 建议 | 改进建议 + 亮点 | 第 6-7 节 |

**"摘要模式"（2 个标签）：**

| 标签 | 内容 | Markdown 段落 |
|-----|---------|-------------------|
| 摘要 | 内容摘要 + 核心观点 | 第 1-2 节 |
| 亮点 | 内容结构 + 亮点 | 第 3-4 节 |

**标签样式：**
- 激活标签：绿色下划线（2px）+ `--text-primary` 颜色
- 非激活标签：`--text-secondary` 颜色，悬停变亮
- 标签切换：交叉淡入淡出动画（150ms）
- 手机上标签栏可水平滚动

### 4. Markdown 渲染增强

报告内部渲染的自定义 CSS：

- **标题：** 正确的层级 — h1 20px, h2 17px, h3 15px。h2 左侧绿色边框。
- **列表：** 自定义项目符号（无序列表绿色圆点，有序列表等宽数字）
- **表格：** 条纹行（偶数行 `--bg-elevated`），适当的单元格内边距，手机上响应式水平滚动
- **粗体文字：** 比正文稍亮
- **评分数值：** 根据数值着色 — ≥7 绿色，4-6 黄色，≤3 红色（解析后通过 CSS 类应用）
- **引用块：** 左侧边框高亮，背景稍暗

### 5. 加载状态

- **骨架屏闪烁卡片** — 加载视频列表时：6 个占位卡片带脉冲动画
- **报告加载** — 获取分析时：展开区域内显示骨架行（3-4 行）
- **按钮加载** — 分析按钮显示"分析中..."文字 + 旋转图标，操作期间禁用
- **标签切换** — 如需重新渲染，显示 200ms 短暂旋转图标

### 6. 动画

| 动画 | 时长 | 缓动 | 触发 |
|-----------|----------|--------|---------|
| 卡片入场 | 300ms + 50ms 交错 | ease-out | 页面/账号加载 |
| 卡片展开 | 300ms | ease-out | 卡片点击 |
| 卡片收起 | 200ms | ease-in | 关闭按钮 / 点击另一张 |
| 标签切换 | 150ms | ease | 标签点击 |
| 状态标签闪烁 | 400ms | ease | 状态更新 |
| 悬停上浮 | 150ms | ease | 鼠标进入 |
| 骨架屏脉冲 | 1.5s | ease-in-out | 加载状态 |

所有动画遵循 `prefers-reduced-motion: reduce` — 禁用变换，使用即时状态切换。

### 7. 页面布局

**左侧边栏（博主列表）：**
- 顶部添加搜索/筛选输入框 — 按昵称筛选博主
- 博主卡片：保持当前样式但添加悬停效果
- 选中的博主：绿色左边框

**右侧内容区域：**
- 粘性头部：文件夹名 + 分析模式选择 + 批量分析按钮
- 滚动视频网格时保持可见

**间距：**
- 网格间距：20px（从 16px 增加）
- 卡片内边距：16px（保留）
- 区域间距：头部和网格之间 24px

## Markdown 标签拆分逻辑

AI 返回单个 markdown 字符串。客户端 JS 通过解析带编号的段落标题（`## 1.`、`## 2.` 等或 `**1.**`、`**2.**` 等）将其拆分为标签页。

**解析方式：**
1. 按顶级段落标记（以 `##` 开头的行或加粗编号标题）拆分 markdown
2. 按索引将段落映射到标签：
   - Full 模式：第 1-2 节 → 摘要，第 3-4 节 → 评分，第 5 节 → 预测，第 6-7 节 → 建议
   - 摘要模式：第 1-2 节 → 摘要，第 3-4 节 → 亮点
3. 如果解析失败（格式不符），回退到在第一个标签中显示全部内容

纯客户端实现 — 不需要后端改动。

## API 变更

无。所有现有接口保持不变。

## 避免的反模式

- 不用 emoji 做图标（用 SVG 或 CSS 形状）
- 不用纯颜色表示状态（标签文字 + 边框颜色双重指示）
- 卡片展开不引起布局偏移（用 transform，不用高度动画影响网格）
- 不阻塞动画（用户可在过渡期间交互）
- 手机上不出现水平滚动（标签在容器内滚动）
